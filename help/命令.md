# 运行命令

本文件集中记录当前工程的正式启动命令和测试命令。所有命令默认在项目根目录执行。

## 正式命令

### 1. 环境准备

- 功能：进入项目虚拟环境，避免误用系统 Python 导致缺少 `torch / pyarrow` 等依赖。
- Windows 命令：

```powershell
. .\.venv\Scripts\Activate.ps1
```

- Linux 命令：

```bash
source .venv/bin/activate
```

- 参数说明：无。
- 注意：如果不激活虚拟环境，也可以直接使用 `.venv\Scripts\python.exe` 运行命令。

### 2. 单卡推理

- 功能：使用单卡或 CPU 加载指定阶段模型，启动终端对话推理。
- 关键参数：
  - `--model`：模型来源，支持 `text_pretrain / chat_sft / chat_lora`，兼容别名 `text_base / lora`。
  - `--execution-mode single`：强制单设备执行；默认也是 `single`。
  - `--lora-base-source`：仅 `chat_lora` 使用，指定 LoRA 基座为 `text_pretrain` 或 `chat_sft`。
- 参数示例：
  - `--model chat_sft`：加载 Chat SFT 整模型权重。
  - `--model chat_lora --lora-base-source text_pretrain`：加载 text pretrain 基座并叠加 LoRA adapter。
- Windows 启动命令：

```powershell
python main.py --model chat_sft --execution-mode single
```

- Linux 启动命令：

```bash
python main.py --model chat_sft --execution-mode single
```

- LoRA 推理命令：

```powershell
python main.py --model chat_lora --lora-base-source text_pretrain --execution-mode single
```

### 3. 单机多卡推理

- 功能：在单进程内按层把模型放到多张可见 GPU 上，用于超过单卡显存承载能力的长上下文推理第一版验证。
- 关键参数：
  - `CUDA_VISIBLE_DEVICES`：控制参与本次推理的物理 GPU。
  - `--execution-mode model_parallel`：启用当前已落地的层级 model parallel。
  - `--execution-mode auto`：当可见 CUDA 卡超过 1 张时自动选择 `model_parallel`，否则回退 `single`。
  - `--device-map auto`：按可见逻辑 GPU 数量和显存自动分配层。
  - `--device-map path/to/device_map.json`：使用手工映射文件覆盖自动切分。
- 参数示例：`CUDA_VISIBLE_DEVICES=0,1 --device-map auto` 表示只使用物理 0、1 两张卡，程序内部自动按逻辑设备 `cuda:0 / cuda:1` 分层。
- Windows 启动命令：

```powershell
$env:CUDA_VISIBLE_DEVICES = "0,1"
python main.py --model chat_sft --execution-mode model_parallel --device-map auto
```

- Linux 启动命令：

```bash
CUDA_VISIBLE_DEVICES=0,1 python main.py --model chat_sft --execution-mode model_parallel --device-map auto
```

- 自动模式命令：

```powershell
$env:CUDA_VISIBLE_DEVICES = "0,1"
python main.py --model chat_sft --execution-mode auto --device-map auto
```

- 手工 device map 示例：

```json
{
  "layers": ["cuda:0", "cuda:0", "cuda:1", "cuda:1"],
  "embedding": "cuda:0",
  "final_norm": "cuda:0",
  "lm_head": "cuda:0"
}
```

- 手工映射启动命令：

```powershell
$env:CUDA_VISIBLE_DEVICES = "2,3"
python main.py --model chat_sft --execution-mode model_parallel --device-map .\device_map.json
```

- 注意：设置 `CUDA_VISIBLE_DEVICES=2,3` 后，程序内部看到的是逻辑设备 `cuda:0 / cuda:1`，手工 `device_map` 中不要写物理 `cuda:2 / cuda:3`。

### 4. 单卡训练

- 功能：运行三阶段训练工作流，当前正式可用路径仍是单进程单卡/CPU。
- 关键参数：
  - `--manifest`：训练数据 manifest。
  - `--eval-manifest`：可选验证数据 manifest，启用周期性或阶段性 `eval_loss / eval_ppl`。
  - `--train-max-sequence-length`：覆盖本次训练样本截断上限。
  - `--longrope2-target-window`：覆盖 LongRoPE2 目标窗口，仅在没有可恢复 checkpoint 时生效。
  - `--longrope2-long-factors-path`：导入 LongRoPE2 因子文件，仅作为导入入口，最终因子会写入 `model_config`。
  - `--longrope2-window-lengths`：训练窗口混合采样长度，例如 `2048,7680`。
  - `--longrope2-window-weights`：训练窗口采样权重，例如 `0.5,0.5`。
- Text pretrain 启动命令：

```powershell
python main-pretrain.py --manifest data/manifests/text_pretrain.json --eval-manifest data/manifests/text_pretrain_eval.json
```

- Chat SFT 启动命令：

```powershell
python main-sft.py --manifest data/manifests/chat_sft.json --eval-manifest data/manifests/chat_sft_eval.json
```

- Chat LoRA 启动命令：

```powershell
python main-LoRA.py --manifest data/manifests/chat_lora.json --eval-manifest data/manifests/chat_lora_eval.json --base-source text_pretrain
```

- LongRoPE2 训练参数示例：

```powershell
python main-sft.py --manifest data/manifests/chat_sft.json --train-max-sequence-length 7680 --longrope2-target-window 32768 --longrope2-window-lengths 2048,7680 --longrope2-window-weights 0.5,0.5
```

### 5. 多卡训练

- 当前状态：训练侧 `FSDP2` 尚未落地，因此当前没有正式可运行的多卡训练命令。
- 目标功能：后续通过 `torchrun` 启动多进程训练，并让进程数匹配 `CUDA_VISIBLE_DEVICES` 暴露的可见 GPU 数量。
- 目标命令形态（当前不可作为已完成命令使用）：

```bash
CUDA_VISIBLE_DEVICES=0,1 torchrun --nproc_per_node=2 main-sft.py --manifest data/manifests/chat_sft.json --eval-manifest data/manifests/chat_sft_eval.json --execution-mode fsdp
```

- 待补参数：`--execution-mode fsdp`、FSDP wrapping policy、distributed checkpoint、rank0 日志/保存策略、optimizer/scheduler 分布式状态恢复。

### 6. 长上下文评测

- 功能：评估指定模型在 needle、长文本 PPL、QA/retrieval 上的长上下文表现。
- 关键参数：
  - `--model`：待评测模型，支持 `text_pretrain / chat_sft / chat_lora`。
  - `--text-manifest`：长文本 PPL 使用的数据 manifest。
  - `--cache-strategy`：缓存策略，当前常用 `session_rebuild`。
  - `--output-format`：输出 `json / markdown / both`。
- 启动命令：

```powershell
python tools/evaluate_long_context.py --model chat_sft --text-manifest data/manifests/text_pretrain.json --cache-strategy session_rebuild --output-format both
```

### 7. LongRoPE2 候选因子 Sweep

- 功能：离线比较 current/bootstrap/uniform/文件导入候选因子对长上下文指标的影响，不写回 checkpoint。
- 关键参数：
  - `--uniform-factor name=value`：构造统一倍率候选，例如 `uniform_8x=8`。
  - `--factors-file name=path`：导入搜索或手工因子文件。
  - `--no-current / --no-bootstrap`：跳过当前 checkpoint 因子或 bootstrap 因子。
- 启动命令：

```powershell
python tools/evaluate_longrope2_factor_sweep.py --model chat_sft --uniform-factor uniform_8x=8 --output-format both
```

### 8. Sequence Packing GPU 基准

- 功能：对比 sequence packing 开关下的真实 GPU 训练吞吐。
- 关键参数：
  - `--manifest`：用于基准的数据 manifest。
  - `--manifest-kind`：manifest 样本类型，支持 `text / chat`。
  - `--packing-mode`：基准模式，支持 `plain / packed / both`。
- 启动命令：

```powershell
python tools/benchmark_sequence_packing.py --manifest data/manifests/text_pretrain.json --manifest-kind text --packing-mode both
```

### 9. 数据转换

- 功能：把原始语料转换为结构化 `text / chat` JSONL，再由 manifest 引用。
- TXT 转 text JSONL：

```powershell
python tools/convert_txt_text_dataset.py --input data/raw/sample.txt --output data/structured/sample.text.jsonl --source-name sample
```

- 原始 text JSONL 标准化：

```powershell
python tools/convert_raw_text_jsonl.py --input data/raw/sample.jsonl --output data/structured/sample.text.jsonl --source-name sample
```

- Instruction/chat JSONL 转结构化 chat：

```powershell
python tools/convert_instruction_chat_jsonl.py --input data/raw/chat.jsonl --sft-output data/structured/chat.sft.jsonl --lora-output data/structured/chat.lora.jsonl --source-name chat
```

## 测试命令

### 1. 编译检查

- 功能：做 Python 语法与导入层面的基础冒烟检查。
- 启动命令：

```powershell
python -m compileall main.py main-pretrain.py main-sft.py main-LoRA.py lpt_config lpt_runtime lpt_data lpt_model lpt_training lpt_inference lpt_lora lpt_protocol lpt_evaluation lpt_workflows tools tests
```

### 2. 执行层与推理回归

- 功能：验证 `CUDA_VISIBLE_DEVICES` 感知、device map、推理 session、模型前向和 checkpoint schema。
- 启动命令：

```powershell
python -m unittest tests.test_runtime_execution tests.test_inference tests.test_model_behavior tests.test_model_config tests.test_checkpoint_schema
```

### 3. 训练 Recipe 回归

- 功能：验证参数分组、梯度累积、验证指标、resume、LongRoPE2 训练策略等训练侧确定性逻辑。
- 启动命令：

```powershell
python -m unittest tests.test_training_recipe
```

### 4. 长上下文与 LongRoPE2 Sweep 回归

- 功能：验证长上下文评测 case 构造、报告落盘、候选因子 sweep 报告格式。
- 启动命令：

```powershell
python -m unittest tests.test_long_context_eval tests.test_longrope2_factor_sweep
```

### 5. 数据管线回归

- 功能：验证结构化 schema、manifest 权重、流式加载、sequence packing 和数据转换工具。
- 启动命令：

```powershell
python -m unittest tests.test_data_pipeline
```

### 6. 全量测试

- 功能：一次性运行 `tests/` 下所有测试。
- 启动命令：

```powershell
python -m unittest discover -s tests
```

- 注意：Windows 本机若遇到 `pyarrow` 原生扩展加载顺序问题，按模块拆开运行，不要把 access violation 误判为模型逻辑失败。

### 7. CLI 入口检查

- 功能：确认主要 CLI 可以正常加载参数帮助。
- 推理入口：

```powershell
python main.py --help
```

- 长上下文评测入口：

```powershell
python tools/evaluate_long_context.py --help
```

- LongRoPE2 sweep 入口：

```powershell
python tools/evaluate_longrope2_factor_sweep.py --help
```
